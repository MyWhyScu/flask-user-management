## 📋 项目简介

简易用户信息管理平台，包含**登录、注册、用户搜索、头像上传、信息展示、登出**功能。  
本项目从**存在多个安全漏洞的初始版本**开始，历经 **五阶段安全加固**，累计修复 **31 项安全隐患**。
其中第四阶段重点修复了 **SQL 注入漏洞**，第五阶段重点修复了 **文件上传漏洞**，是本次课程的核心内容。

### 📌 SQL注入修复详情（第四阶段核心）

**漏洞代码（修复前）：**
```python
# ❌ f-string字符串拼接 — 存在SQL注入
sql = f"SELECT ... WHERE username LIKE '%{keyword}%'"
c.execute(sql)
```

**修复代码（修复后）：**
```python
# ✅ 参数化查询 — 安全
sql = "SELECT ... WHERE username LIKE ?"
c.execute(sql, (f'%{keyword}%',))
```

**修复原理：** 参数化查询将SQL语句与用户数据**分离编译**，数据库先编译SQL模板，再将参数作为纯数据传入。即使用户输入 `' OR '1'='1`，数据库也只将其当作**普通搜索文本**，而非SQL代码。

## ⚙️ 技术栈

- **后端**: Python Flask
- **数据库**: SQLite 3.46.1
- **数据库交互**: sqlite3 原生驱动（已全面使用参数化查询）
- **密码存储**: Werkzeug `generate_password_hash` (PBKDF2-SHA256)
- **日志**: Python `logging` 模块（记录所有登录/注册事件）
- **前端**: Jinja2 模板 + 原生 CSS

### 安全修复涉及的知识点

| 知识点 | 对应漏洞 |

| 认证安全 | 密码哈希、Session 管理 |
| 注入防护 | 参数化查询、输入校验、CSP |
