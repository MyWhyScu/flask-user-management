## 📋 项目简介

简易用户信息管理平台，包含登录、用户信息展示、登出功能。  
本项目从**存在多个安全漏洞的初始版本**开始，逐步修复了 **21 项安全隐患**，最终达到安全可用的状态。

## 🚀 快速开始

### 环境要求

- Python 3.8+
- Flask
- Werkzeug

### 安装依赖

```bash
pip install flask werkzeug
```

### 生成 SSL 证书（HTTPS 需要）

```bash
mkdir -p certs
openssl req -x509 -newkey rsa:2048 \
  -keyout certs/key.pem \
  -out certs/cert.pem \
  -days 365 -nodes \
  -subj "/CN=你的IP地址"
```

### 启动服务

```bash
python app.py
```

### 访问地址

| 方式 | 地址 | 说明 |
|------|------|------|
| **HTTPS 主站** | `https://127.0.0.1:5000` | 加密传输 |
| HTTP 自动跳转 | `http://127.0.0.1:5080` | 自动跳转到 HTTPS |
| 局域网访问 | `https://你的局域网IP:5000` | 同网段设备可访问 |

## 🛡️ 安全修复清单

### 第一阶段：严重/高危漏洞修复（7项）

| # | 漏洞类型 | 风险等级 | 修复措施 |
|---|----------|----------|----------|
| 1 | Debug模式远程代码执行 | 🔴 严重 | `debug=False` 关闭 Werkzeug Debugger |
| 2 | Session 密钥硬编码 | 🔴 严重 | `os.urandom(24).hex()` 生成强随机密钥 |
| 3 | 明文存储密码 | 🔴 严重 | `werkzeug.security` 哈希存储 + 哈希比对 |
| 4 | HTML注释泄露账号 | 🟠 高危 | 删除登录页中的调试注释 |
| 5 | 密码显示在页面 | 🟠 高危 | 密码字段不再传递给模板 |
| 6 | 监听所有网卡 | 🟠 高危 | 保留 `0.0.0.0`（支持跨机访问） |
| 7 | Session Cookie 缺安全标志 | 🟠 高危 | 添加 `HttpOnly`、`SameSite`、`Secure` |

### 第二阶段：中危漏洞修复（8项）

| # | 漏洞类型 | 修复措施 |
|---|----------|----------|
| 8 | 无审计日志 | `logging` 记录所有登录事件 |
| 9 | 无登录频率限制 | 内存限流器：每分钟每 IP 最多 5 次 |
| 10 | 无 CSRF 保护 | `secrets.token_hex(32)` 令牌验证 |
| 11 | 无输入校验 | 正则校验 + 长度限制 |
| 12 | 缺安全响应头 | 添加 CSP、X-Frame-Options 等 5 项 |
| 13 | 服务器版本泄露 | Server 响应头清空 |
| 14 | 用户名枚举 | 统一错误提示 + 固定延时防时序攻击 |
| 15 | CSRF 登出（GET） | `/logout` 改为 POST only |

### 第三阶段：HTTPS 加密（1项）

| # | 漏洞类型 | 修复措施 |
|---|----------|----------|
| 16 | HTTP 明文传输 | 启用 HTTPS + HTTP→HTTPS 自动跳转 |

## ⚙️ 技术栈

- **后端**: Python Flask
- **密码存储**: Werkzeug `generate_password_hash` (PBKDF2-SHA256)
- **日志**: Python `logging` 模块
- **HTTPS**: OpenSSL 自签名证书
- **前端**: Jinja2 模板 + 原生 CSS

## 📝 课程作业说明

本项目的初始版本刻意保留了常见的安全漏洞，用于教学演示。  
经过三阶段安全加固，展示了 Web 安全实践中常见问题的识别与修复方法。

### 安全修复涉及的知识点

- 信息泄露防护（注释、版本号、错误信息）
- 认证安全（密码哈希、Session 管理、HTTPS）
- 注入防护（输入校验、CSP）
- 会话保护（CSRF Token、HttpOnly、SameSite）
- 暴力破解防护（频率限制、统一错误提示）
- 日志审计
- 安全响应头配置
